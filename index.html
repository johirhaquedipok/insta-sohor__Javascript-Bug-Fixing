FAQ

JavaScript is a client-side scripting language. It runs on the client-side inside the web-browser. The web browser has a JavaScript engine that runs the code. For example, Chrome has a JavaScript engine called V8. AS JavaScript is an interpreted language. It gets executed line by line. The JavaScript engine converts the code line by line to byte code (machine language) and runs the code.

The JavaScript engine starts to execute a script from the top of the line. It creates an execution context. It pushes a function into the call stack and pops it out when it's done. But for asynchronous tasks, the JavaScript engine pushes the task into a queue called a callback queue. The Event Loop constantly checks both the callback queue and the call stack. When the call stack is empty, the event loop places the task from the callback queue to the call stack.

Data stored in local storage does not expire. But data in Session Storage gets cleared when the browser tab is closed.

Data that are not confidential but reduce the amount of traffic between client and server is saved in Local Storage. But any data that can be confidential use the Session Storage so that can be cleared out after the user closes the browser or window.
